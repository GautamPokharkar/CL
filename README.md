## Experiment 1 – IAM Users and Groups

1. **What is IAM in AWS?**
   IAM is Identity and Access Management used to control AWS access.

2. **Why are IAM groups used?**
   Groups simplify permission management for multiple users.

3. **What is a managed policy?**
   A managed policy is a predefined permission set by AWS.

4. **What is MFA?**
   MFA adds extra security using a second authentication method.

5. **Why enforce MFA?**
   MFA prevents unauthorized account access.

6. **What permission does S3-Support have?**
   Read-only access to Amazon S3.

7. **What permission does EC2-Admin have?**
   View, start, and stop EC2 instances.

8. **What is least privilege principle?**
   Giving only required permissions to users.

## Experiment 2 – EBS Volume and Snapshot

1. **What is Amazon EBS?**
   EBS is block storage for EC2 instances.

2. **Why attach EBS to EC2?**
   To provide persistent storage to the instance.

3. **What is an EBS snapshot?**
   A backup copy of an EBS volume.

4. **Can snapshots restore data?**
   Yes, new volumes can be created from snapshots.

5. **What file system is commonly used on EBS?**
   ext4 is commonly used in Linux.

6. **Is EBS persistent after EC2 stop?**
   Yes, data remains stored.

7. **Why mount a volume?**
   To make storage accessible to the OS.

## Experiment 3 – AWS S3 Storage

1. **What is Amazon S3?**
   S3 is an object storage service.

2. **What is an S3 bucket?**
   A container used to store objects/files.

3. **What types of files can S3 store?**
   Images, videos, documents, and backups.

4. **What is a public object in S3?**
   A file accessible through the internet.

5. **What is the use of S3 URL?**
   To access uploaded files online.

6. **Is S3 scalable?**
   Yes, it provides unlimited storage scalability.

7. **What is object storage?**
   Storage where data is stored as objects/files.

## Experiment 4 – Amazon RDS

1. **What is Amazon RDS?**
   RDS is a managed relational database service.

2. **Which databases are supported in RDS?**
   MySQL, PostgreSQL, MariaDB, Oracle, etc.

3. **Why use RDS?**
   It automates backups, updates, and scaling.

4. **What is DB instance class?**
   It defines CPU and memory capacity.

5. **Why enable backups?**
   To recover data during failures.

6. **What is public access in RDS?**
   Allows remote database connectivity.

7. **What is SQL used for?**
   To manage and query relational databases.

## Experiment 8 – VPC Configuration

1. **What is a VPC?**
   Virtual Private Cloud is a private AWS network.

2. **What is a public subnet?**
   Subnet with internet access.

3. **What is a private subnet?**
   Subnet without direct internet access.

4. **What is an Internet Gateway?**
   Allows internet connectivity to VPC.

5. **What is a NAT Gateway?**
   Provides internet to private subnet resources.

6. **Why use multiple Availability Zones?**
   For high availability and fault tolerance.

7. **What is route table?**
   Defines network traffic routing rules.
